Overview

StayFresh is a static archive of workflow research and patterns for AI-assisted development. Content is organized into workflows (practical patterns) and research (empirical findings).

The immediate job is straightforward: keep the site static, keep GitHub Pages publishing it, and put Anubis in front so scrapers do not get a free buffet.

Thesis

StayFresh Thesis is the site's center of gravity: the hard position, the named frameworks, and the shortest path through the archive.

Core rule: tools should reduce comprehension debt. Auditory cues are one practical way to improve recall when attention is split across terminals, editors, and agent loops.

GitHub Pages + Anubis

GitHub Pages cannot run Anubis. It serves files. That is the whole deal.

The sane setup is: keep Pages as the static origin, put a real reverse proxy in front of your custom domain, and point that proxy at the GitHub Pages origin instead of back at yourself.

visitor
  -> caddy or nginx on your server
  -> anubis
  -> https://your-user.github.io or https://your-org.github.io/your-repo

Notes

Cloudflare is simpler for most people. Anubis is the move when you want the control and you are willing to own the box in front of the box.

If you point Anubis at your public custom domain instead of the GitHub Pages origin, you create a stupid proxy loop and buy yourself a bad evening.
